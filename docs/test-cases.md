# 回归测试用例

这些用例覆盖当前 PWA MVP 的核心能力。每次提交前都应执行：

```bash
npm run verify
```

本仓库已配置本地 git `pre-commit` hook，在当前 clone 中提交前会自动运行同一套校验。

## 本轮已发现并修复的问题

| ID | 问题 | 修复方式 |
| --- | --- | --- |
| BUG-001 | 底部导航被主内容层遮挡，真实点击无法切换页面 | 调整底部导航层级，确保点击命中导航按钮 |
| BUG-002 | 孕中途进入的用户会被过去事项直接淹没，难以判断哪些已做、哪些需补救 | 增加历史事项 swipe review deck，让用户先逐项左划“已完成”或右划“需补救” |
| BUG-003 | 单个提醒标题包含非法文件名字符时，日历导出文件名不稳定 | 导出前统一清洗文件名 |
| BUG-004 | 主内容滚动时底部导航会跟着内容跑，整体不像手机 app | 改为固定底部 dock，主内容单独滚动，并兼顾安全区留白 |

## 用例清单

| 用例 ID | 场景 | 关键步骤 | 预期结果 | 自动化 |
| --- | --- | --- | --- | --- |
| TC-001 | 首次进入完成建档 | 填写伴侣昵称、预产期、城市、医院并保存 | 如果用户处于孕期中段，会先进入历史事项 review deck，再回到聚焦式首页 | 是 |
| TC-002 | 历史事项左右滑 review | 对历史任务做左滑和右滑 | 左滑记为已完成，右滑进入“需补救”，动效和状态都正确 | 是 |
| TC-003 | 底部导航切换 | 从仪表盘滚动内容，再点击“清单”“提醒”“仪表盘” | 底部 dock 保持固定可见，三个主视图都可真实点击切换，标题同步变化 | 是 |
| TC-004 | 新增自定义任务 | 在清单页填写标题、备注、分类、优先级并提交，然后刷新 | 新任务可见，刷新后仍保留，可手动删除 | 是 |
| TC-005 | 任务移到以后 | 在“当前要做”里把任务执行“移到以后” | 任务从当前分组移出，并能在“以后再说”里找到 | 是 |
| TC-006 | 完成任务与筛选 | 在清单页勾选任务完成，再切到“已完成”筛选 | 已完成列表能看到刚完成的任务 | 是 |
| TC-007 | 提醒导出 | 在提醒页点击“导出全部” | 浏览器下载 `.ics` 文件，内容包含日历头、医院地点和任务摘要 | 是 |
| TC-008 | 编辑资料 | 打开资料弹层，修改医院并保存 | 提醒页中的节点地点同步更新为新医院 | 是 |
| TC-009 | 本地持久化 | 完成建档、添加任务后刷新页面 | 用户资料和自定义任务仍存在 | 被 TC-004 覆盖 |

## 执行方式

```bash
npm run test:e2e
```

如果需要完整提交前校验，执行：

```bash
npm run verify
```
